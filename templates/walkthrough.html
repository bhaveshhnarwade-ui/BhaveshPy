Running the App
# 1. Install Flask (one-time)
pip install flask

# 2. Start the server
python App.py

# 3. Open in your browser
http://localhost:5000

The SQLite database menu.db is created automatically on first run. No manual setup needed.

Order Flow
StepWhat Happens
1. Enter nameBrowser asks for GPS location silently in background
2. Range checkCoords sent to /api/check_delivery — if >15 km, show out-of-range screen
3. Browse menuMenu loads normally — no ETA shown yet
4. Add items & checkoutCart sent to /api/bill with distance_km
5. ReceiptShows bill + ETA breakdown + Leaflet map with delivery route
Delivery System

The ETA is calculated server-side in App.py using constants from config.py. It only appears on the receipt — after the customer places an order.

ComponentValueWhy
Kitchen Prepavg of ordered items + 10 minReflects the actual items ordered, plus a 10 min cooking buffer
Travel4 min/km~15 km/h average city speed with traffic & signals
Handoff Buffer5 minDriver finding house / parking
Distance Limit15 kmBeyond this → out-of-range screen before menu loads

Formula: ETA = (avg_item_prep + 10) + distance_km × 4 + 5

To change server location before a demo: open config.py, update SERVER_LAT and SERVER_LNG, restart Flask.

Delivery Route Map (Leaflet + OSM)

After an order is placed, the receipt modal shows a live Leaflet + OpenStreetMap map — no API key required.

ElementDetails
🍽 Gold pinRestaurant — Bhaskar Mali Chawk, Bhiwandi (from config.py)
🏠 Blue pinCustomer's GPS location
Dashed gold lineDelivery route polyline connecting both points
Dark popupsCustom-styled to match the luxury theme

The server returns server_lat and server_lng in the bill response — the frontend uses these to place the restaurant pin without hardcoding coordinates in JS.
